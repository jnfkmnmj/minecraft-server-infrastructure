## Case Studies

This repository documents the complete process of deploying, maintaining, and troubleshooting a modded Minecraft Forge server on Ubuntu Linux.

The following case studies summarize major engineering challenges encountered during the project.

| Case Study | Description |
|------------|-------------|
| [Cloud Server Provisioning](docs/case-studies/cloud-server-provisioning-case-study.md) | Selecting cloud infrastructure, deploying Ubuntu, installing Java, and successfully launching the first Paper server. |
| [Forge Deployment Retrospective](docs/case-studies/forge-deployment-retrospective.md) | Deploying a Forge server, resolving startup failures, adapting community solutions, and validating the deployment workflow. |
| [Mod Integration and Compatibility](docs/case-studies/mod-integration-and-compatibility.md) | Systematically diagnosing compatibility issues among Forge mods and establishing a stable mod combination. |
| [TACZ Gunpack Customization](docs/case-studies/tacz-gunpack-customization.md) | Debugging gunpack loading behavior, identifying active resource paths, and successfully customizing weapon configurations. |
| [Performance and Stability Analysis](docs/case-studies/performance-and-stability-analysis.md) | Monitoring resource usage, analyzing server performance, and improving runtime stability for a large Forge modpack. |
| [Server Migration and Backup Strategy](docs/case-studies/server-migration-and-backup-strategy.md) | Preserving the complete server environment, migrating between systems, and validating backup integrity before retiring the original VPS. |


These documents describe real engineering problems encountered during the project rather than isolated tutorials. Each case study focuses on the investigation process, technical reasoning, and final solution.
